# Host Backend 개선 포인트

- [x] `HostBackendState`가 구체 타입인 `InputController`를 역참조하여 컨트롤러와 백엔드 사이에 양방향 결합이 있다.
- [x] 백엔드가 실제 `InputController` 없이는 생성되기 어려워 백엔드 단위 테스트와 테스트 대역 구성이 어렵다.
- [x] `busyAppsSnapshot`이 전역 가변 상태이며, 애플리케이션 시작 과정의 암묵적인 초기화 순서에 의존한다.
- [x] `hostBackend`가 implicitly unwrapped optional이어서 초기화되지 않은 상태를 타입으로 배제하지 못한다.
- [x] 백엔드 선택과 생성 책임이 `InputController` 초기화 코드에 결합되어 있다.
- [x] 두 백엔드에 키 처리, 입력 모드 전환, Roman 입력, lifecycle 처리 등 공통 동작이 중복되어 있어 구현이 서로 달라질 수 있다.
- [x] 두 백엔드가 공통으로 만족해야 하는 동작 계약이 자동 검사로 명확하게 고정되어 있지 않다.
- [x] 실제 백엔드 선택, 콜백 위임, lifecycle 순서를 직접 검증하는 자동 검사가 부족하다.
- [x] `HostBackend`가 호스트 통합 동작 전체를 하나의 큰 전략으로 묶고 있어 서로 독립적인 호환성 차이를 별도로 표현하기 어렵다.
- [x] 호환성 요구가 늘어나면 백엔드 종류와 중복 구현이 함께 증가할 가능성이 있다.
